Endpoints dealing with user signup

Set Profile Details (step 1/3 of signup)

First step of the signup process. Validates profile details.

Request Body schema: application/json
required

Profile registration details.

email
required
string <email> ^[^\s@]+@[^\s@]+\.[^\s@]+$

User's unique email address

first_name
required
string >= 2 characters ^[A-Za-z'\- ]+$

User's first name (letters only, min 2 chars)

last_name
required
string >= 2 characters ^[A-Za-z'\- ]+$

User's last name (letters only, min 2 chars)

profession
string
Enum: "professional" "researcher" "analyst" "technician" "consultant"

User's profession

Set Password (step 2/3 of signup)

Allows a user to define a secure password as part of the registration flow. Password must meet strength requirements.

Request Body schema: application/json
required

Password to set.

password
required
string >= 8 characters ^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[^A-Za-z0...

Min 8 characters, at least 1 uppercase, 1 lowercase, 1 number, 1 symbol

session_id
required
string <uuid>

Unique identifier of the signup session

Responses

/signup/password

Finalize Signup (step 3/3 of signup)

Final step of the signup process, creating a new user. User must accept T&C and can optionally accept marketing communications.

Request Body schema: application/json
required

T&C acceptance and marketing consent.

accept_terms
required
boolean

Must be true to accept Terms & Conditions and proceed with registration

accept_marketing
boolean

Optional consent for marketing communications

session_id
required
string <uuid>

Unique identifier of the signup session

Simple authentication with refresh tokens

Login

Validates user login credentials, returns a JWT access token and a refresh token. Very simple auth, acceptable for POC, not suitable for production.

Request Body schema: application/json
required
email
required
string <email>

User's email address

password
required
string >= 8 characters

User's password (minimum 8 characters)

Responses

/auth/login

}

Response samples

  • 200
  • 401
Content type
application/json
{
  • "access_token": "string",
  • "refresh_token": "string",
  • "expires_in": 1
}
